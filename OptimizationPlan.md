# Phase 4: System Optimization & Scaling Plan

## Overview
This document outlines the optimization strategy for the LILIT Chatbot SaaS monolithic backend. Because the application is hosted on Render's Free Tier (512MB RAM), these optimizations are required to prevent Out of Memory (OOM) crashes, protect API billing limits, and handle concurrent student traffic efficiently.

**Execution Order:**
1. Rate Limiting (Security & Billing Protection)
2. Serverless Vector DB (RAM & Storage Protection)
3. Widget Minification (Frontend Performance)
4. Asynchronous Database (Concurrency)

---

## 1. Security: API Rate Limiting (Priority: High)
**Goal:** Prevent malicious users or bots from draining the master OpenAI/Gemini API key by spamming the public `/chat` endpoint.

**Dependencies:**
```text
# Add to requirements.txt
slowapi
Implementation Steps:

Open server.py.

Import the slowapi library and initialize the Limiter using the client's IP address.

Attach the limiter to the FastAPI app state.

Wrap the @app.post("/chat") and @app.post("/chat/bot/{bot_id}") routes with the @limiter.limit("20/minute") decorator.

Update the JavaScript catch block in the widget (inside the embed generation and index.html) to display a user-friendly "Too many messages. Please wait a moment." when a 429 status code is returned.

2. Memory: Pinecone Migration (Priority: High)
Goal: Eliminate the local tenant_dbs/ FAISS folders. Offload 100% of vector storage and RAM usage to a free cloud database to prevent Render server crashes.

Dependencies:

Plaintext
# Add to requirements.txt
pinecone-client
langchain-pinecone

Implementation Steps:

Create a free account on Pinecone.io and get an API key. Add PINECONE_API_KEY to the Render Environment Variables.

Open server.py and replace all FAISS imports with from langchain_pinecone import PineconeVectorStore.

Update Uploads (/upload-docs & /scrape-url): Instead of calling faiss_db.save_local(), initialize the PineconeVectorStore and pass namespace=website_id. Once uploaded to Pinecone, use Python's os.remove() to immediately delete the original PDF from the Render server.

Update Queries (/chat/{website_id}): Replace the FAISS.load_local() logic with a Pinecone retriever, filtering by namespace=website_id.

3. Performance: Widget Minification (Priority: Medium)
Goal: Reduce the payload size of the dynamic JavaScript generated by the /embed/{bot_id}.js endpoint so client websites load the chatbot instantly.

Implementation Steps:

Open server.py.

Add a minify_js(code: str) helper function at the top of the file that uses the re (Regex) module to strip out newlines (\n), multi-line comments (/* ... */), and excess whitespace.

In the @app.get("/embed/{bot_id}.js") endpoint, pass the massive js_code string through the minify_js function right before returning the Response.

4. Concurrency: Asynchronous Database (Priority: Low)
Goal: Upgrade the standard SQLAlchemy synchronous database calls to async so the FastAPI event loop isn't blocked when dozens of students chat simultaneously.

Dependencies:

Plaintext
# Add to requirements.txt
sqlalchemy[asyncio]
aiosqlite
Implementation Steps:

Create a new file called database.py to separate the database connection logic from server.py.

Update the SQLite connection string to use the async driver: sqlite+aiosqlite:///./lilit.db.

Replace sessionmaker with AsyncSessionLocal and use create_async_engine.

In server.py, update the dependency injection Depends(get_db) to yield an AsyncSession.

Update all database queries in the endpoints to use await (e.g., await db.execute(select(Chatbot).filter_by(bot_id=bot_id))).
